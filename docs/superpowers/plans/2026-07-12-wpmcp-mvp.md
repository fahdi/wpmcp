# wpmcp MVP Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Ship an MCP-Adapter-native WordPress plugin where an AI agent builds/edits pages, with snapshot-before-every-write + one-click rollback as the trust wedge — free tier complete, Pro/Freemius hooks wired from day one.

**Architecture:** A single WordPress plugin (PHP 8.1+) riding the official `WordPress/mcp-adapter` + Abilities API. Every mutating tool routes through one `Safe_Mutation` orchestrator that snapshots the WordPress data layer (post content, builder meta, options) before applying, verifies after, and can roll back one operation or a whole agent session. Builder-agnostic core; Gutenberg editing ships in the free MVP, Elementor deep-edit is the Phase 2 paid hook.

**Tech Stack:** PHP 8.1, WordPress 6.9+, Composer (PSR-4 + dev tooling), PHPUnit 9 + wp-phpunit + Yoast polyfills, GitHub Actions, Freemius SDK, GPL-2.0-or-later.

## Global Constraints

- **License:** GPL-2.0-or-later. `LICENSE` file present; every PHP file has an `ABSPATH` guard.
- **Namespacing:** PHP namespace root `WPMCP\` → `src/` (PSR-4). No global functions except the Freemius helper `wpmcp_fs()` and template tags. Text domain: `wpmcp`.
- **Versions:** WordPress ≥ 6.9, PHP ≥ 8.1. Do not use APIs newer than these floors.
- **Safety-first invariant:** No mutating MCP tool may write to the DB except through `WPMCP\Safety\Safe_Mutation::run()`. This is the product promise and is enforced by tests.
- **Non-destructive defaults:** Write tools that can destroy data ship disabled-by-default and require `confirm:true`. Reads are always safe.
- **Pro gating from day one:** Every ability is tagged `free` or `pro`; `WPMCP\Pro\Gate` decides registration/limits. Free tier caps snapshot history at 20 operations; Pro is unlimited.
- **Open source:** Public git repo, conventional-commit messages, no AI-attribution trailers in commits.
- **TDD:** Red → green → commit for every task. No implementation without a failing test first.

```php
// src/Safety/Snapshot.php
<?php
namespace WPMCP\Safety;
class Snapshot {
    public static function capture( string $object_type, int $object_id ): array {
        $post = get_post( $object_id, ARRAY_A );
        return [
            'object_type' => $object_type,
            'object_id'   => $object_id,
            'data'        => [
                'post' => $post ? [ 'post_content' => $post['post_content'], 'post_title' => $post['post_title'], 'post_status' => $post['post_status'] ] : null,
                'meta' => get_post_meta( $object_id ),
            ],
        ];
    }
    public static function serialize( array $before ): string {
        return gzencode( wp_json_encode( $before ) );
    }
    public static function unserialize( string $blob ): array {
        return json_decode( gzdecode( $blob ), true );
    }
}
```

```php
    return (int) $wpdb->query( $wpdb->prepare( "DELETE FROM {$t} WHERE id <= %d", $cutoff ) );
}
```

```php
// src/Safety/Safe_Mutation.php
<?php
namespace WPMCP\Safety;
class Safe_Mutation {
    public static function run( array $context, callable $mutation, ?callable $verify = null ): array {
        $operation_id = wp_generate_uuid4();
        $snapshot     = Snapshot::capture( $context['object_type'], (int) $context['object_id'] );
        Snapshot_Store::save(
            $operation_id,
            $context['session_id'],
            $snapshot,
            $context['tool_name'],
            hash( 'sha256', wp_json_encode( $context['args'] ?? [] ) )
        );
        $result = $mutation();
        if ( $verify && ! $verify( $result ) ) {
            self::restore( $snapshot );
```

```php
// src/Safety/Rollback_Service.php
<?php
namespace WPMCP\Safety;
class Rollback_Service {
    public static function restore_operation( string $operation_id ): bool {
        $row = Snapshot_Store::get_by_operation( $operation_id );
        if ( ! $row ) { return false; }
        self::apply_snapshot( $row['snapshot'] );
        return true;
    }
    public static function restore_session( string $session_id ): int {
        $rows = Snapshot_Store::list_by_session( $session_id ); // newest first
        $rows = array_reverse( $rows ); // oldest first, so we can unwind to the earliest
        // Restore the OLDEST snapshot per object (its pre-session state).
        $seen = [];
        $count = 0;
        foreach ( $rows as $r ) {
            $key = $r['object_type'] . ':' . $r['object_id'];
            if ( isset( $seen[ $key ] ) ) { $count++; continue; }
            $seen[ $key ] = true;
            self::apply_snapshot( Snapshot::unserialize( $r['before_blob'] ) );
            $count++;
        }
        return $count;
    }
    public static function apply_snapshot( array $snapshot ): void {
        if ( 'post' === $snapshot['object_type'] && $snapshot['data']['post'] ) {
            wp_update_post( array_merge( [ 'ID' => $snapshot['object_id'] ], $snapshot['data']['post'] ) );

        }
```
